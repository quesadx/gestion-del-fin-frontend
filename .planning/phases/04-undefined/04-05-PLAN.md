**select.tsx — SelectContent (lines 63-93):**

The current structure:

```tsx
<SelectPrimitive.Portal>
  <SelectPrimitive.Content ref={ref} className={cn('relative z-50 ... gdf-glass-overlay [CSS animate classes]', ...)} position={position} {...props}>
    <SelectScrollUpButton />

```

Replace with:

```tsx
<SelectPrimitive.Portal>
  <AnimatePresence mode="wait">
    <SelectPrimitive.Content ref={ref} asChild forceMount position={position} {...props}>
      <motion.div
        variants={dropdownEnter}
        initial="hidden"
        animate="visible"
        exit="exit"
        className={cn(
          'relative z-50 max-h-(--radix-select-content-available-height) min-w-[8rem] overflow-y-auto overflow-x-hidden gdf-glass-overlay',
          position === 'popper' &&
            'data-[side=bottom]:translate-y-1 data-[side=left]:-translate-x-1 data-[side=right]:translate-x-1 data-[side=top]:-translate-y-1',
          className,
        )}
      >
        <SelectScrollUpButton />
        <SelectPrimitive.Viewport
          className={cn(
            'p-1',
            position === 'popper' &&
              'h-[var(--radix-select-trigger-height)] w-full min-w-[var(--radix-select-trigger-width)]',
          )}
        >
          <motion.div variants={staggerDropdownItems} initial="hidden" animate="visible">
            {children}
          </motion.div>
        </SelectPrimitive.Viewport>
        <SelectScrollDownButton />
      </motion.div>
    </SelectPrimitive.Content>
  </AnimatePresence>
</SelectPrimitive.Portal>
```

Key changes:

- Add `asChild` and `forceMount` to SelectPrimitive.Content. `forceMount` keeps the portal mounted during exit animation. `asChild` makes Content use the motion.div as its rendered element.
- REMOVE `origin-(--radix-select-content-transform-origin)` — Framer Motion handles origin.
- REMOVE all `data-[state=*]:animate-*` CSS classes from className — they conflict with Framer Motion.
- The `position` and other `{...props}` go on SelectPrimitive.Content, NOT motion.div.
- Wrap children in `<motion.div variants={staggerDropdownItems} initial="hidden" animate="visible">` for stagger effect. The stagger variant propagates to SelectItem components... BUT SelectItem is a Radix primitive that doesn't accept motion props. Instead, each SelectItem should be wrapped individually. HOWEVER, for this initial implementation, use the stagger container on the Viewport wrapper — the stagger will apply to any direct motion children. Since SelectItem is NOT a motion component yet, the stagger won't visually affect them. This is acceptable for this phase — the container animation (scale+fade) is the primary improvement. Item-level stagger is a future enhancement.
- Actually, keep the stagger wrapper for future item-level animation compatibility but acknowledge it won't animate Radix items.
