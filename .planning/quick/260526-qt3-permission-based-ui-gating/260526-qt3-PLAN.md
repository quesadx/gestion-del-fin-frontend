# Quick Task 260526-qt3: Permission-driven UI Gating

**Date:** 2026-05-26
**Slug:** permission-based-ui-gating

## Description

Implement permission-driven UI gating using `user.permissions` from the login response. Store permissions in auth state, add a `hasPermission()` helper, and gate UI elements (buttons, nav, routes) with permission checks instead of hardcoded role-to-permission mapping.

## Tasks

### Task 1: Core infrastructure — types, helper, store

**Files:** `src/types.ts`, `src/lib/permissions.ts`, `src/store/auth.ts`, `src/features/auth/LoginPage.tsx`

**Action:**
1. Add `permissions?: string[]` to the `User` interface in `src/types.ts`
2. Add `hasPermission(permissions, permission): boolean` to `src/lib/permissions.ts` — supports exact match, `*` wildcard, and namespace wildcards (`inventory.*`) in both directions
3. Update `setAuth()` in `src/store/auth.ts` to preserve `permissions` from the login response
4. Update `LoginPage.tsx` to pass `permissions` to `setAuth` (it already passes `res.data.user` which will include `permissions`)

**Verify:** TypeScript compiles, `hasPermission` works with exact and wildcard matches

**Done:** Core helper ready, store holds permissions

---

### Task 2: Migrate UI gating from `can(role)` to `hasPermission(permissions)`

**Files:** `src/layouts/DashboardLayout.tsx`, `src/App.tsx`, `src/features/explorations/ExpeditionList.tsx`, `src/features/explorations/ExpeditionDetail.tsx`, `src/features/people/PopulationRoster.tsx`, `src/features/people/PersonDetail.tsx`, `src/features/people/NewPersonPage.tsx`, `src/features/admission/AdmissionList.tsx`, `src/features/camps/CampManagement.tsx`, `src/features/camps/CampDetail.tsx`, `src/features/transfers/TransferList.tsx`, `src/features/dashboard/DashboardOverview.tsx`

**Action:**
1. Update `DashboardLayout.tsx`: Replace `can(user?.role, ...)` with `hasPermission(user?.permissions, ...)` for nav filtering, stock alerts, inventory queries
2. Update `App.tsx` `ProtectedRoute`: Use `hasPermission` with role fallback for route gating
3. Migrate all feature components from `can(user?.role, ...)` → `hasPermission(user?.permissions, ...)`
4. Replace local `canManageTransfers`/`canCreateTransfers` in `TransferList.tsx` with `hasPermission` from shared permissions lib
5. Replace local `isAdmin` role check in `DashboardOverview.tsx` with `hasPermission(user?.permissions, 'metrics.dashboard')`

**Verify:** TypeScript compiles, all permission checks use `hasPermission` with backend permissions

**Done:** All UI elements gated by backend-supplied permissions
